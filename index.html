          Declarativo

          React faz com que a criação de UIs interativas seja uma tarefa fácil. Crie views simples para cada estado na sua aplicação, e o React irá atualizar e renderizar de forma eficiente apenas os componentes necessários na medida em que os dados mudam.

          Views declarativas fazem com que seu código seja mais previsível e simples de depurar.

          Baseado em componentes

          Crie componentes encapsulados que gerenciam seu próprio estado e então, combine-os para criar UIs complexas.

          Como a lógica do componente é escrita em JavaScript e não em templates, você pode facilmente passar diversos tipos de dados ao longo da sua aplicação e ainda manter o estado fora do DOM.

          Aprenda uma vez, use em qualquer lugar

          Não fazemos suposições sobre as outras tecnologias da sua stack, assim você pode desenvolver novos recursos com React sem reescrever o código existente.

          O React também pode ser renderizado no servidor, usando Node, e ser usado para criar aplicações mobile, através do React Native.

        React é uma biblioteca ou uma framework?

        • React é frequentemente referido como uma biblioteca JavaScript para construir interfaces de usuário. Foi criado pelo Facebook e é mantido por eles junto com uma comunidade de desenvolvedores independentes e empresas.
        • React foca principalmente na criação de componentes de UI que podem ser reutilizados em diferentes partes de uma aplicação web ou móvel. No entanto, ele também pode ser considerado como uma framework em alguns aspectos, especialmente quando combinado com um ecossistema completo de ferramentas de roteamento, gerenciamento de estado e interações com APIs, como Redux, React Router, entre outros.
        • A principal distinção entre uma biblioteca e uma framework é que uma biblioteca fornece funções específicas para serem chamadas pelo usuário, enquanto uma framework fornece um esqueleto ou estrutura onde o usuário encaixa seu código. React se encaixa mais na definição de biblioteca porque não impõe uma estrutura rígida ao desenvolvedor e oferece mais liberdade na construção da aplicação.

        O que é JSX?

        • JSX é uma extensão de sintaxe para JavaScript recomendada pelo React. Ela se parece muito com XML ou HTML e é usada dentro de arquivos JavaScript para descrever a estrutura da interface do usuário. O JSX permite escrever elementos HTML em JavaScript e colocá-los no DOM sem usar createElement e appendChild ou similares.
        • Aqui está um exemplo de JSX:
          const element = <h1>Hello, world!</h1>;
        • Este código define uma constante element que contém um elemento JSX que descreve um cabeçalho HTML. O uso de JSX não é obrigatório para usar React, mas é uma conveniência visual que muitos desenvolvedores preferem devido à sua legibilidade e familiaridade com a marcação HTML.
        • Quando o JSX é processado, ele é transformado em chamadas de funções JavaScript. Essas chamadas criam objetos que o React usa para construir o DOM virtual. Em tempo de compilação, o JSX é transformado em chamadas para React.createElement. Portanto, o código JSX acima é convertido em algo parecido com isto:
          const element = React.createElement('h1', null, 'Hello, world!');
        • Essa transformação é feita geralmente por um transpiler como Babel. O JSX é uma parte fundamental do desenvolvimento com React porque proporciona uma maneira expressiva de definir árvores de componentes e seus elementos filhos.

        O QUE É JAVASCRIPT

        JavaScript é uma linguagem de programação interpretada, amplamente utilizada para o desenvolvimento web. Aqui estão algumas características e usos principais do JavaScript:

        1. Linguagem de Scripting: JavaScript é uma linguagem de scripting que pode ser embutida em páginas HTML. É executada pelo navegador web, permitindo a criação de interatividade e funcionalidades dinâmicas nas páginas da web.
        1. Interpretação em Tempo Real: JavaScript é interpretado pelo navegador, o que significa que o código é executado diretamente pelo navegador, sem a necessidade de uma etapa de compilação.
        1. Desenvolvimento Front-End: JavaScript é essencial para o desenvolvimento front-end, sendo usado junto com HTML e CSS para criar interfaces de usuário interativas. Bibliotecas e frameworks populares como React, Angular e Vue.js são construídos sobre JavaScript.
        1. Desenvolvimento Back-End: Com o surgimento do Node.js, JavaScript também pode ser usado para o desenvolvimento de servidor (back-end), permitindo que desenvolvedores usem a mesma linguagem tanto no cliente quanto no servidor.
        1. Manipulação do DOM: JavaScript permite a manipulação do Document Object Model (DOM), que representa a estrutura HTML de uma página web. Isso permite a atualização de conteúdo e a resposta a interações do usuário sem a necessidade de recarregar a página.
        1. APIs e Serviços Web: JavaScript é frequentemente usado para fazer requisições HTTP para APIs e serviços web, facilitando a comunicação entre o cliente e o servidor e a integração com serviços de terceiros.
        1. Mobile e DESKTOP: Pode ser feito apps mobiles e aplicativos para desktop também.

        JavaScript é uma linguagem versátil, com uma ampla comunidade de desenvolvedores e uma vasta quantidade de bibliotecas e frameworks que facilitam o desenvolvimento de aplicações web complexas e interativas.

        • CONSOLE.LOG()

          O console.log é um método muito utilizado em JavaScript para imprimir informações no console do navegador ou em um ambiente de execução como o Node.js. Ele é fundamental para depuração (debugging) e desenvolvimento de código, pois permite que os desenvolvedores vejam o valor de variáveis e o fluxo de execução do programa.

          Como Usar o console.log

          Aqui está um exemplo básico de como usar o console.log:

          console.log("Hello, World!");

          Quando este código é executado, a string "Hello, World!" será exibida no console.

          Exemplos de Uso

          1. Imprimir Variáveis:
            let nome = "João";
console.log(nome); // Imprime: João
          1. Imprimir Objetos e Arrays:
            let pessoa = { nome: "Maria", idade: 30 };
console.log(pessoa); // Imprime: { nome: "Maria", idade: 30 }

let numeros = [1, 2, 3, 4, 5];
console.log(numeros); // Imprime: [1, 2, 3, 4, 5]
          1. Mensagens Combinadas:
            let idade = 25;
console.log("A idade é:", idade); // Imprime: A idade é: 25
          1. Depuração de Código:
            function somar(a, b) {
    console.log("Valores recebidos - a:", a, ", b:", b);
    return a + b;
}

let resultado = somar(5, 3);
console.log("Resultado da soma:", resultado); // Imprime: Resultado da soma: 8

          Variações do console

          Além do console.log, o objeto console oferece outros métodos úteis:

          • console.error(): Para imprimir mensagens de erro.
            console.error("Ocorreu um erro!");
          • console.warn(): Para imprimir avisos.
            console.warn("Este é um aviso.");
          • console.table(): Para imprimir arrays e objetos em formato de tabela.
            let usuarios = [
    { nome: "Ana", idade: 28 },
    { nome: "Bruno", idade: 32 }
];
console.table(usuarios);
          • console.info(): Para imprimir mensagens informativas.
            console.info("Informação importante.");
          • console.debug(): Para imprimir mensagens de depuração (geralmente oculto por padrão).
            console.debug("Mensagem de depuração.");

          Uso no Node.js

          No Node.js, o console.log funciona da mesma forma, mas o console é a saída padrão do terminal.

          Conclusão

          O console.log é uma ferramenta essencial no desenvolvimento com JavaScript. Ele ajuda a entender o comportamento do código, rastrear variáveis e depurar problemas. É uma prática comum utilizar console.log durante o desenvolvimento e remover ou comentar essas linhas antes de enviar o código para produção, para evitar a poluição do console do usuário final.

        • A TAG SCRIPT

          A tag <script> é uma parte fundamental do HTML e é usada para incorporar scripts de linguagem de programação, como JavaScript, em documentos HTML. Esses scripts podem ser usados para adicionar interatividade, controlar o comportamento do navegador e dinamicamente modificar o conteúdo da página.

          Estrutura Básica

          A estrutura básica da tag <script> é a seguinte:

          <script>
  // Código JavaScript aqui
</script>

          Atributos Comuns

          1. src: Especifica o URL de um arquivo de script externo.
            <script src="script.js"></script>
          1. type: Especifica o tipo de script. O valor padrão é text/javascript.
            <script type="text/javascript">
  // Código JavaScript aqui
</script>
          1. async: Executa o script de forma assíncrona em relação ao restante da página (apenas para scripts externos).
            <script src="script.js" async></script>
          1. defer: Deferra a execução do script até que a página tenha sido completamente carregada (apenas para scripts externos).
            <script src="script.js" defer></script>

          Posição da Tag <script> no HTML

          1. Dentro do <head>:
            Colocar a tag
            <script> dentro do <head> é comum quando você deseja que o script seja carregado antes do conteúdo da página. No entanto, isso pode causar atrasos no carregamento da página se o script for grande.
            <head>
  <script src="script.js"></script>
</head>
          1. No Final do <body>:
            Colocar a tag
            <script> no final do <body> é uma prática recomendada para garantir que o conteúdo HTML seja carregado antes do script ser executado.
            <body>
  <!-- Conteúdo da página -->
  <script src="script.js"></script>
</body>

          Uso de Scripts Inline vs. Externos

          • Inline: Scripts inline são definidos diretamente dentro da tag <script>.
            <script>
  alert('Olá, Mundo!');
</script>
          • Externos: Scripts externos são definidos em arquivos separados, que são incluídos usando o atributo src.
            <script src="script.js"></script>

          Asynchronous vs. Deferred Scripts

          • async: Scripts marcados com async são carregados em paralelo ao carregamento da página e executados assim que são carregados. Isso pode ser útil para scripts que não dependem de outros scripts ou do conteúdo da página.
            <script src="script.js" async></script>
          • defer: Scripts marcados com defer são carregados em paralelo, mas a execução é adiada até que a página tenha sido completamente carregada. Isso garante que o script não bloqueie a renderização da página.
            <script src="script.js" defer></script>

          Uso Comum

          1. Manipulação do DOM:
            <script>
  document.addEventListener('DOMContentLoaded', function() {
    document.getElementById('myElement').innerText = 'Texto atualizado pelo JavaScript';
  });
</script>
          1. Interatividade:
            <button onclick="alert('Botão clicado!')">Clique em mim</button>
          1. Validação de Formulários:
            <form onsubmit="return validateForm()">
  <input type="text" id="name" />
  <input type="submit" value="Enviar" />
</form>
<script>
  function validateForm() {
    var name = document.getElementById('name').value;
    if (name === '') {
      alert('Nome é obrigatório');
      return false;
    }
    return true;
  }
</script>

          Conclusão

          A tag <script> é essencial para adicionar funcionalidades dinâmicas e interativas às páginas da web. Com a capacidade de incluir scripts inline ou externos, além de controlar o comportamento de carregamento com async e defer, os desenvolvedores têm a flexibilidade necessária para otimizar e melhorar a experiência do usuário.

        • Uma boa prática é colocar nossos códigos javascritpt em arquivos separados.
        • Se a tag script tiver o src preenchido ele não executa o código dentro dessa tag script.
        • Podemos colocar nossa tag script no final do body para fazer com que seu carregamento seja no final.
        • Podemos colocar no header, mas interpretador do navegador irá carregar e executar seu código primeiro, o que pode ocasionar bugs na página que está sendo carregada.
        • Temos o atributo defer na tag script, isso faz com que ela carregue primeiro o js em paralelo com a página, mas só executa o conteúdo no final do carregamento da página.

        • NODE JS

          Node.js é um ambiente de execução JavaScript baseado no motor V8 do Google Chrome, que permite a execução de código JavaScript no lado do servidor. Criado por Ryan Dahl em 2009, Node.js é conhecido por sua eficiência, escalabilidade e desempenho, especialmente em aplicações de rede.

          Características Principais

          1. Orientado a Eventos e Não Bloqueante:
            • Node.js utiliza um modelo de I/O não bloqueante e orientado a eventos, o que o torna eficiente e escalável, especialmente para aplicações que fazem uso intensivo de entrada/saída (I/O), como servidores web e APIs.
          1. Motor V8 do Google Chrome:
            • O motor V8 compila código JavaScript para código de máquina nativo, oferecendo alta performance.
          1. Único Thread com Event Loop:
            • Node.js opera em um único thread com um loop de eventos, permitindo o gerenciamento eficiente de múltiplas conexões simultâneas sem a necessidade de criar múltiplas threads.
          1. Gerenciamento de Pacotes com npm:
            • O Node Package Manager (npm) é o gerenciador de pacotes padrão para Node.js, permitindo a instalação e gerenciamento de bibliotecas e ferramentas JavaScript.

          Usos Comuns

          1. Servidores Web:
            • Node.js é frequentemente usado para criar servidores web, graças à sua eficiência e capacidade de lidar com um grande número de conexões simultâneas.
            const http = require('http');

const server = http.createServer((req, res) => {
  res.statusCode = 200;
  res.setHeader('Content-Type', 'text/plain');
  res.end('Hello World\\n');
});

server.listen(3000, '127.0.0.1', () => {
  console.log('Server running at <http://127.0.0.1:3000/>');
});
          1. APIs RESTful:
            • A criação de APIs RESTful é um caso de uso comum para Node.js, muitas vezes em conjunto com frameworks como Express.js.
            const express = require('express');
const app = express();

app.get('/', (req, res) => {
  res.send('Hello World');
});

app.listen(3000, () => {
  console.log('Server is running on port 3000');
});
          1. Aplicações em Tempo Real:
            • Node.js é ideal para aplicações que exigem comunicação em tempo real, como chats e jogos online.
            const http = require('http');
const socketIo = require('socket.io');
const server = http.createServer();
const io = socketIo(server);

io.on('connection', (socket) => {
  console.log('a user connected');
  socket.on('disconnect', () => {
    console.log('user disconnected');
  });
});

server.listen(3000, () => {
  console.log('Listening on port 3000');
});
          1. Ferramentas de Linha de Comando:
            • Node.js é frequentemente usado para construir ferramentas de linha de comando e scripts de automação.
            #!/usr/bin/env node
console.log('Hello from Node.js');

          Frameworks e Bibliotecas Populares

          1. Express.js: Um framework minimalista e flexível para construir aplicações web e APIs.
          1. Koa.js: Criado pela mesma equipe do Express, Koa é mais moderno e modular.
          1. NestJS: Um framework para construir aplicações de servidor escaláveis e eficientes, utilizando TypeScript.
          1. Socket.io: Biblioteca para comunicação em tempo real.
          1. Mongoose: Biblioteca de modelagem de dados para MongoDB.

          /*
  Este é um comentário de múltiplas linhas.
  Pode ser usado para descrever partes mais complexas do código
  ou para adicionar documentação extensa.

let y = 10;

  console.log(chave + ': ' + pessoa[chave]);
}

        Objetos Aninhados

        Objetos podem conter outros objetos, permitindo estruturas de dados complexas.

        let empresa = {
  nome: 'TechCorp',
  endereco: {
    rua: 'Av. Principal',
    numero: 123,
    cidade: 'São Paulo'
  }
};

console.log(empresa.endereco.cidade); // São Paulo

        Propriedade this

        Dentro de um método, this refere-se ao objeto ao qual o método pertence.

        let pessoa = {
  nome: 'João',
  saudacao: function() {
    console.log('Olá, meu nome é ' + this.nome);
  }
};

pessoa.saudacao(); // Olá, meu nome é João

        Manipulação de Propriedades

        Você pode adicionar, modificar e deletar propriedades de um objeto.

        let carro = {
  marca: 'Toyota',
  modelo: 'Corolla'
};

// Adicionando uma propriedade
carro.ano = 2020;

// Modificando uma propriedade
carro.marca = 'Honda';

// Deletando uma propriedade
delete carro.modelo;

        Object Methods

        JavaScript oferece vários métodos úteis para trabalhar com objetos:

        1. Object.keys(obj) - Retorna um array com as chaves do objeto.
          let chaves = Object.keys(pessoa); // ['nome', 'idade', 'profissão']
        1. Object.values(obj) - Retorna um array com os valores do objeto.
          let valores = Object.values(pessoa); // ['João', 31, 'Arquiteto']
        1. Object.entries(obj) - Retorna um array de arrays contendo pares de chave-valor.
          let entradas = Object.entries(pessoa); // [['nome', 'João'], ['idade', 31], ['profissão', 'Arquiteto']]
        1. Object.assign(target, ...sources) - Copia as propriedades de um ou mais objetos fonte para um objeto alvo.
          let destino = {};
let fonte1 = {a: 1};
let fonte2 = {b: 2};
Object.assign(destino, fonte1, fonte2); // destino agora é {a: 1, b: 2}

        Destructuring

        A desestruturação permite extrair valores de objetos de forma concisa.

        let pessoa = {
  nome: 'João',
  idade: 31,
  profissão: 'Arquiteto'
};

let { nome, idade } = pessoa;
console.log(nome); // João
console.log(idade); // 31
